Managed Identity

Overview ├втАУ┬╝

  • What it is ├втВмтАЭ an identity automatically managed by Microsoft Entra ID that can be assigned to an Azure resource (Container App, VM, App Service, Automation Account, etc.) so the resource's code can authenticate without storing credentials.
  • System-assigned vs. user-assigned ├втВмтАЭ system-assigned identities are tied 1:1 to a resource and deleted with it; user-assigned identities are standalone resources that can be shared across multiple resources.
  • Local identity endpoint ├втВмтАЭ when a managed identity is assigned, the resource's runtime exposes IDENTITY_HEADER / IDENTITY_ENDPOINT (or MSI_ENDPOINT / MSI_SECRET on older platforms) env vars, used to request tokens locally.
  • Token = identity ├втВмтАЭ any code (or shell) running on the resource can request a token for the identity for any resource URI (ARM, Microsoft Graph, Key Vault, Storage, etc.) it has been granted access to.
  • Pivoting ├втВмтАЭ a token for https://management.azure.com can be used with Connect-AzAccount -AccessToken ... -AccountID ... to operate as the identity in Az PowerShell, inheriting whatever RBAC roles it holds.
  • Find the Account ID ├втВмтАЭ the Account ID is the identity's principalId, visible on the resource (e.g. via az resource show / Get-AzResource ├втВмтАЭ look for the Identity field).
