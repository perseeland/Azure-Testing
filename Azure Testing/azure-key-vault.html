Azure Key Vault

Overview ▼

  • What it is — a managed service for securely storing and accessing secrets (API keys, passwords, connection strings, tokens, certificates, etc.).
  • Access methods — Azure RBAC controls who can list/read/delete secrets via the ARM management plane; the Key Vault REST API (with an access token) allows reading secret values.
  • Managed identity integration — if a Container App, VM, or Automation Account has a managed identity with Key Vault access, it can retrieve secrets via the REST API without hardcoding credentials.
  • Network access — Key Vaults can be configured for public access, private endpoint, or firewall rules; enumerate firewall rules and allowed subnets for possible bypass.
  • Secrets at risk — once you have an access token (via managed identity or user/service principal credentials), you can list all secrets and retrieve their values via the REST API.
